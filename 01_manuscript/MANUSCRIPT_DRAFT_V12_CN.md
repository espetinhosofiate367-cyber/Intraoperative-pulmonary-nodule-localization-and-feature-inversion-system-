# 基于触觉感知与深度学习的术中肺结节定位系统：一项前沿性探索研究

## 摘要
微创胸腔镜和机器人辅助手术显著降低了肺结节切除的创伤，但也使术者失去了开放手术中的直接触觉反馈，从而增加了小结节、深部结节和隐匿性结节的术中探测与定位难度。围绕这一临床问题，本文以**可靠术中探测**为首要目标，并进一步探索结节大小反演和粗深度辨别的可行性，提出并验证了一条由柔性触觉感知、层级深度学习、可解释性分析和部署增强共同构成的技术路线。研究首先在离体猪肺模型上采集由 7 个结节大小、6 个埋藏深度和 3 次重复构成的时序触觉数据，并通过机制分析提取扩散、形态、分布复杂度和时序阶段等结构化物理特征。随后，利用 XGBoost 建立结构化基线，验证触觉数据中同时存在稳定的结节探测信息、强大小信息以及可学习的粗深度信息。在此基础上，本文进一步构建**仅输入原始时空张量**的层级神经网络：阶段一执行结节检测，阶段二执行大小分类与连续回归，阶段三在大小条件下执行粗深度分类，并通过 latent probe、hard-pair analysis 和 phase occlusion 检验网络是否自动编码了与大小和深度相关的物理结构。实验结果表明，阶段一检测模型在独立测试集上的 AUC 为 0.8383，高于 XGBoost 的 0.8199；优化后的 raw-input size-only 模型在真实阳性窗口上的 top-1、top-2 和 MAE 分别达到 0.7177、0.8195 和 0.1242，已全面超过 XGBoost 的 0.6701、0.8023 和 0.1472；在深度任务上，原始输入网络若不显式处理 size-depth 耦合会退化至多数类附近，而在 size-aware 与 route-aware 组织下，纯 raw-input 的 route-aware 深度模型将 predicted-route 平衡准确率提升至 0.5240，超过 XGBoost 的 0.5138，并将 ground-truth route 提升至 0.6066。进一步的解释性分析表明，raw-input 神经网络内部确实编码了部分与大小和深度相关的物理结构，而非仅依赖单点峰值强度。最后，本文将结构化物理特征作为部署增强分支加入统一层级反演器，使真实 predicted-route 深度平衡准确率进一步提升至 0.5337。整体来看，本文更适合作为一项面向术中肺结节定位的前沿性探索研究：其价值不在于宣称已完成临床定型系统，而在于以扎实的数据协议、清晰的任务组织和可检验的证据链，证明了柔性触觉时空数据能够首先支持可靠探测，并进一步支持大小反演和粗深度辨别。

**关键词**：肺结节探测；柔性触觉阵列；术中导航；大小反演；粗深度分类；可解释人工智能

## 1 引言
随着低剂量计算机断层扫描在肺癌早期筛查中的普及，临床上对微小肺结节和磨玻璃结节的检出率持续升高。对于此类疑似早期病灶，电视辅助胸腔镜手术和机器人辅助手术因其创伤小、恢复快和住院周期短，已成为重要的外科干预方式。然而，微创手术在带来创伤获益的同时，也引入了一个尚未被彻底解决的临床难题：术者在依赖细长器械操作时，几乎完全失去了开放手术中可直接用于识别病灶的触觉反馈。对于深部、小尺寸或隐匿性结节，这种触觉缺失会显著增加漏检、定位不准和扩大切除范围的风险。

目前临床上常见的辅助定位方案，包括术前 CT 引导钩线或染料定位、术中超声以及有限度的数字触诊，但这些方法均存在明确短板。术前侵入式定位具有气胸、出血和标记移位等风险；术中超声对探头贴合、空气干扰控制和操作者经验依赖较强；数字触诊虽然直观，但高度主观，缺乏可量化输出。已有综述和回顾性研究已较充分总结了这些路线在胸外科场景中的价值与局限 [R1-R3]。因此，术中定位真正需要的并不是另一个“辅助视觉手段”，而是一种能够在微创条件下恢复缺失触觉、并将其转化为客观定量信息的技术体系。

在微创肿瘤定位方向，已有机器人辅助触觉感知、力反馈融合、声反射或谐振式触觉传感以及相关综述工作证明，深部包块可以通过机械响应差异被感知和定位 [R4-R8]。这些工作为术中触觉恢复提供了重要启发，但从我们对现有文献的综合判断来看，既有研究大多聚焦于单一定位任务、相对理想化的组织模型或单一路径验证，而较少在肺部特异场景下，将“可靠探测、大小反演、粗深度辨别、raw-input 学习与可解释性验证”组织为同一条层级证据链。

柔性阵列触觉传感器为这一目标提供了有前景的路径。与传统单点力传感器不同，柔性阵列可以在按压过程中连续记录肺表面的二维应力分布，并形成具有时间演化特征的时空触觉张量。当高刚度结节埋藏在软组织内部时，结节与周围组织的力学不匹配会改变表面应力场的峰值、扩散范围、空间形态以及时间响应模式。理论上，这些表面信号中同时包含三个层次的信息：首先是**结节是否存在**，其次是**结节大小**，最后才是更弱、更难的**埋藏深度**。

然而，将表面触觉分布直接反演为内部结节参数，本质上是一个病态逆问题。一个浅层但较小的结节，可能在某一按压时刻产生与一个深层但较大的结节相似的表面热点；如果只依赖单帧应力图或简单幅值阈值，模型极易将大小效应误写成深度效应，反之亦然。因此，本文不将“检测、大小、深度”视作同等难度的平行输出，而是将其组织为一个由易到难、由强信号到弱信号的层级问题：**先回答有没有结节，再回答结节有多大，最后在大小条件下探索深度是否能够被粗粒度辨别。**

基于这一认识，本文采用“**机制分析 -> 结构化基线 -> raw-input 学习 -> 可解释性验证 -> 部署增强**”的研究路径。首先，通过统计与物理启发特征分析回答数据中是否存在与探测、大小和深度相关的结构化信号；其次，利用 XGBoost 建立结构化基线，验证这些信号是否足以支撑检测、大小反演和粗深度分类；随后，构建**不直接输入结构化物理特征**的 raw-input 层级神经网络，让模型从原始时空触觉窗口中自动学习这些信息；在此基础上，再通过可解释性分析检验网络是否确实捕捉到了与大小和深度相关的部分物理特征；最后，出于系统部署目的，将结构化物理特征加入统一层级反演器，用于提升真实 predicted-route 条件下的系统级鲁棒性。

需要强调的是，本文引入 XGBoost 并不是为了与神经网络形成简单替代关系。XGBoost 在本研究中承担三个角色。第一，它是**结构化机制基线**，用于回答“数据里是否真实存在可学习的信息”；第二，它是**解释锚点**，帮助识别与深度相关的扩散、形态、分布复杂度和时序阶段特征家族；第三，它是**结构化参照**，使我们能够区分“神经网络是否真正从原始输入中学到了更多”与“神经网络是否只是复用了结构化先验”。与此同时，XGBoost 路线依赖显式窗口级物理特征提取，这使其更适合作为机制分析与离线参照，而不天然适合作为 detection-first 的实时主线。

因此，本文希望回答的不是“神经网络是否全面替代结构化方法”，而是四个更具体的问题：
1. 柔性触觉时空数据能否首先支持稳定的术中结节探测？
2. 在探测基础上，是否可以可靠反演结节大小，并进一步粗略区分深度？
3. 在不直接输入结构化物理特征的情况下，raw-input 神经网络能否从原始时空张量中自动学习这些信息？
4. 当 raw-input 神经网络达到或超过结构化基线后，它内部是否确实编码了部分与大小和深度相关的物理结构？

围绕上述问题，本文的主要贡献可以概括为：
1. 建立了面向术中肺结节触觉探测与反演的结构化时空数据体系。在 7 个结节大小、6 个埋藏深度和 3 次重复实验条件下，获得 126 组逐帧应力分布序列，并据此建立与检测、大小反演和粗深度分类相对应的窗口级标签协议。
2. 通过机制分析和 XGBoost 基线，证明了柔性触觉时空数据中既存在稳定的结节探测信息，也包含强大小信息和可学习的粗深度信息，从而为后续神经网络建模提供结构化参照。
3. 提出了以检测为核心的 raw-input 层级神经网络体系，将任务组织为“Detection -> Size -> Depth”，并证明原始时空张量本身足以支持可靠探测、较强大小反演及大小条件下的粗深度辨别。
4. 通过 latent probes、hard-pair analysis 与 phase occlusion，建立了从结构化特征、XGBoost 解释到 raw-input 神经网络内部表征的可解释性闭环。
5. 在 scientific main line 之外，进一步构建面向部署的统一层级反演器，通过融合原始时空输入与结构化物理特征并显式优化多种 route 条件，提高真实 predicted-route 场景下的系统级深度性能，并验证该方向具备实时界面集成潜力。

## 2 材料与方法

### 2.1 实验平台与数据协议
本研究使用柔性阵列触觉传感系统对离体猪肺模型进行动态按压实验。每一帧包含 96 个传感通道，经重排后形成 12×8 的二维应力矩阵。实验设计矩阵由 7 个结节大小（0.25、0.5、0.75、1.0、1.25、1.5、1.75 cm）和 6 个埋藏深度（0.5、1.0、1.5、2.0、2.5、3.0 cm）构成，共 42 个物理条件；每个条件进行 3 次独立重复，最终得到 126 组带有完整时间演化信息的逐帧触觉序列。

为适配模型输入，本文采用长度为 10 帧、步长为 2 的滑动窗口作为基本样本单位。检测任务以窗口中心帧是否位于人工标注阳性片段内作为标签；大小和深度标签继承自对应实验记录的物理条件。检测任务使用 `1.CSV + 2.CSV` 作为开发集、`3.CSV` 作为独立测试集；大小与深度反演任务采用更严格的重复划分协议，即 `1.CSV` 训练、`2.CSV` 验证、`3.CSV` 测试，并仅在真实阳性窗口上训练。

### 2.2 数据表示与窗口级输入组织
针对每个 10 帧窗口，本文同时保留三类互补表示。第一，`raw amplitude` 用于保留绝对幅值、整体能量和峰值水平；第二，`normalized shape` 通过逐帧归一化强调热点形态、扩散和中心-边缘对比；第三，`delta sequence` 用于表示相邻阶段的变化趋势与时序斜率。这种输入组织的出发点是，术中触觉并不是单一静态纹理识别问题，而是一个同时依赖绝对强度、空间形态和动态变化的时空感知问题。若只保留其中一种表示，模型要么容易过分依赖峰值，要么容易丢失与大小或阶段相关的重要信息。

### 2.3 机制分析与结构化物理特征
在进入深度学习之前，本文首先对帧级和窗口级触觉特征进行系统分析。候选特征包括峰值强度、积分响应、中心-边缘对比、热点面积、热点半径、二阶矩扩散、空间熵、质心位置及漂移、峰值到达时间、衰减特征以及多种窗口级时序统计量。分析结果表明，结节大小是最强主效应，主要影响峰值、整体能量和中心集中度；深度则是更弱但条件依赖的效应，主要体现为扩散范围、分布复杂度、形变位置和按压阶段响应。基于此，本文将结构化物理特征大致归纳为 amplitude、spread extent、shape contrast、deformation position 和 temporal phase 五类。

### 2.4 XGBoost 结构化基线
在机制分析的基础上，本文采用 XGBoost 分别构建检测、大小分类/回归和深度粗分类基线。该基线不承担最终部署角色，而是用于回答两个基础问题：其一，这些结构化特征是否足以支撑检测、大小反演和粗深度分类；其二，哪些特征家族在深度判别中真正起主导作用。与直接进入神经网络相比，结构化基线能更早回答“信息是否存在且可学”，从而避免把全部结论建立在黑箱模型之上。与此同时，XGBoost 的特征贡献结果也为后续神经网络设计提供了结构化约束，尤其是在深度主要依赖 spread、shape contrast、deformation position 和 temporal phase，而非单一峰值幅值这一点上。

### 2.5 纯 raw-input 层级神经网络

#### 2.5.1 阶段一：结节探测
检测是本文的首要任务。阶段一检测器直接从原始时空触觉窗口中判断“当前窗口下方是否存在结节”。为此，本文采用双流时空网络，对原始窗口和相邻帧差分窗口并行编码；空间维通过轻量卷积提取，时间维通过 MS-TCN 聚合，最终输出结节存在概率。该阶段的目标不是反演全部物理参数，而是先建立稳定可靠的术中探测器，为后续反演提供高置信入口。

#### 2.5.2 阶段二：大小反演
在真实阳性窗口上，系统进一步执行大小反演，输出 7 类大小分类和连续大小回归。由于前期分析表明大小是强主效应，且也是深度任务的路由条件，本文为大小任务单独训练 raw-input size-only router v2，而不是将其简单附属于共享多任务头。该模型通过 `size classification + ordinal size prediction + expectation-based continuous estimation + residual correction` 的组合，使大小学习既保留档位判断能力，又具备档内连续修正能力。

#### 2.5.3 阶段三：大小条件下的粗深度分类
深度任务被定义为 `shallow / middle / deep` 三类粗分类，而非连续深度回归。更重要的是，本文并未采用全局共享的深度头，而是使用 raw-input size-routed depth experts。其核心假设是：深度效应并非独立于大小存在，而是在不同大小条件下表现出不同的扩散和时序模式。因此，深度必须在大小条件下建模，而不是与大小平级并列输出。进一步地，本文在第二轮优化中显式引入 route-aware 训练，同时优化 `GT route`、`hard predicted route`、`soft route` 和 `top2 route`，使深度专家不只在理想路由下有效，也能适应真实部署中的路由不确定性。

#### 2.5.4 设计理由
本文最关键的架构设计并不是网络更深，而是把深度从“共享头中的又一个分类任务”改写为“大小条件下的 routed experts”。这一步的本质是把物理问题直接写进模型结构：当大小是强主效应而深度是弱条件效应时，深度不应被要求在全局共享特征上直接分离，而应在大小语境下学习局部决策边界。正是这一点，使得 raw-input 神经网络可以在不直接输入结构化物理特征时恢复 coarse depth。

### 2.6 raw-input 神经网络的可解释性分析
为了回答 raw-input 神经网络是否真正学到了与大小和深度相关的物理结构，本文对阶段三 size-routed 深度网络执行三类解释性分析：
1. **latent linear probes**：冻结训练后的 raw-input 深度网络，在其内部潜在表示上训练线性 probe，以恢复前期机制分析中识别出的结构化物理特征，检验网络是否自动编码了这些概念；
2. **hard-pair analysis**：构造“大而深 / 小而浅”等容易混淆的样本对，观察模型是否仍能依据扩散、形态和时序差异做出更合理判断；
3. **phase occlusion**：按按压阶段遮挡输入，检验模型的时间策略是否主要依赖峰值邻域，还是已充分利用 loading / release 等阶段信息。

### 2.7 面向部署的统一层级反演器
在 scientific main line 之外，本文还构建了部署导向的统一层级反演器。该模型联合输入原始幅值序列、归一化形态序列、差分序列以及筛选后的结构化物理特征，并同时输出 size classification、size ordinal prediction、size regression 和 routed depth。与早期“先训 size，再外接 depth expert”的松耦合串联方式不同，统一层级反演器在训练中显式加入 ground-truth route、hard predicted route、soft predicted route、top2 route 和 route consistency 目标，从而直接优化真实 predicted-route 条件下的系统级深度性能。需要强调的是，该模型的定位是**部署增强模型**，而不是用于证明“神经网络完全依赖 raw-input 自动学出所有物理结构”的主证据。

### 2.8 系统落地与实时推理
在工程实现上，检测结果始终显示，大小和深度结果仅在检测概率超过阈值后输出，并最终集成到术中实时可视化界面中。对于 raw-input scientific model，我们关注其在不引入结构化先验时的学习能力与可解释性；对于统一层级反演器，我们关注其在真实部署链路中的 route-aware 鲁棒性与实时显示能力。

## 3 结果

### 3.1 机制分析与 XGBoost 基线先证实了“可探测、可反演、可粗分深度”的信息层级
结构化特征分析首先表明，结节是否存在、结节大小和埋藏深度在触觉数据中的表现强度并不相同。其中，检测相关差异最显著，大小是强主效应，而深度是更弱、更容易被大小掩盖的条件依赖信号。XGBoost 基线进一步将“信息存在”转化为“信息可学”：在测试集上，XGBoost 的检测 AUC 为 0.8199，大小反演达到 top-1 0.6701、top-2 0.8023 和 MAE 0.1472 cm，而深度粗分类平衡准确率达到 0.5138，显著高于多数类基线的 0.3333。这一结果说明，深度并非完全不可学，只是相比检测和大小更弱、更依赖正确的任务组织。

### 3.2 纯 raw-input 神经网络在检测任务上已经超过结构化基线
在独立测试集 `3.CSV` 上，阶段一 raw+delta 检测器取得 AUC 0.8383、AP 0.5357 和 F1 0.6216，整体优于 XGBoost 的 0.8199、0.5063 和 0.6185。对本文而言，这一结果的意义不仅在于分数提升，更在于它证明了：在不依赖手工设计特征的情况下，原始时空张量本身已经足以支持稳定术中探测。

### 3.3 纯 raw-input 神经网络已经在大小任务上全面超过 XGBoost
在真实阳性窗口上，优化后的 raw-input size-only router v2 的 top-1、top-2 和 MAE 分别达到 0.7177、0.8195 和 0.1242，优于 XGBoost 的 0.6701、0.8023 和 0.1472 cm。这个结果说明，大小不仅是强信号，而且原始时空张量本身已经足以支撑同时兼顾分类与连续回归的大小学习。更重要的是，这一提升是在**不直接输入结构化物理特征**的前提下获得的，因此它直接强化了本文 scientific main line 的核心论点：原始时空触觉张量本身已经包含足以超过结构化基线的大小信息。

### 3.4 若不显式处理 size-depth 耦合，raw-input 深度网络会退化到多数类附近
最初的 raw-input end-to-end 深度模型在严格重复划分协议下表现接近多数类基线，测试平衡准确率约为 0.33。这一结果并不意味着原始时空触觉数据不能支持深度学习，而是更清楚地揭示了本文的核心困难：当大小是强主效应、深度是弱主效应时，如果仍采用普通共享 depth head，网络会优先吸收强度和大小相关信号，从而使深度判别被更强因素淹没。

### 3.5 在 size-aware 与 route-aware 组织下，raw-input 神经网络已经可以学出 coarse depth，并超过 XGBoost
当单一深度头被替换为 size-routed depth experts 后，情况发生显著变化。在第一轮 size-aware 组织下，raw-input size-routed 深度网络的 ground-truth route 测试平衡准确率达到 0.5238，已经与 XGBoost 的 0.5138 相当并略有超过。进一步地，当本文将优化后的 raw-input size router 接入训练，并显式对 `GT route / hard predicted route / soft route / top2 route` 进行 route-aware 训练后，纯 raw-input 路线的 ground-truth route 平衡准确率进一步提高到 0.6066，而 predicted-route 平衡准确率提升至 0.5240，首次稳定超过 XGBoost 的 0.5138。这一结果表明：深度不是不可学，而是必须在 `size-aware + route-aware` 架构下建模。

### 3.6 可解释性结果表明，raw-input 神经网络确实自动编码了部分与大小和深度相关的物理结构
为了验证 raw-input 神经网络不是仅凭黑箱相关性做出判断，本文进一步对 size-routed 深度网络开展解释性分析。冻结该模型后，在其潜在表示上训练线性 probe 恢复结构化物理特征，得到 latent probe 的平均测试 `R^2 = 0.3657`，明显高于仅以 size identity 为输入的 `size-only probe (R^2 = 0.2356)`。这一结果说明，raw-input 网络内部已经自动编码了部分超出大小标签本身的深度相关结构。

进一步地，hard-pair analysis 显示，在构造的“大而深 / 小而浅”等困难样本对中，更深样本在约 75% 的配对中获得更高的 `p_deep`，表明模型并非单纯依赖峰值幅值完成判断。phase occlusion 则显示，当前 raw-input 深度网络在时间维上的主要依赖仍集中在 `peak neighborhood`，而 `loading early` 与 `release` 的贡献相对较弱。这意味着 raw-input 神经网络已经自动捕捉到部分与大小和深度相关的物理特征，但尚未完全恢复机制分析所提示的全部时序结构。

### 3.7 机制分析、XGBoost 与 raw-input 神经网络共同形成了完整的科学证据链
综合 3.1-3.6 的结果可以看到，本文已经形成一条较完整的科学证据链：机制分析先指出候选物理特征家族，XGBoost 再证明这些结构化特征足以支撑检测、大小反演和粗深度分类，而 raw-input 神经网络随后在不直接输入这些结构化物理特征的情况下，仍然能够在检测任务、大小任务和 predicted-route 深度任务上超过 XGBoost。最后，raw-input 神经网络的解释性分析又进一步说明，它内部确实自动编码了与 XGBoost 一致的一部分物理结构。这条证据链回答的，是“原始时空张量本身能否学习到这些信息”这一科学问题。

### 3.8 真正阻碍部署的不是“深度信息不存在”，而是 `size -> depth` 路由误差
尽管第一版 raw-input size-routed 深度网络在 oracle route 条件下表现良好，但真实部署必须使用预测 size route。早期链路中，旧 size router 在 Stage3 正窗上的 size top-1 约为 0.6422，predicted-route 深度平衡准确率下降至 0.4822，说明 route mismatch 仍然是主要误差传播源。针对这一问题，本文进一步从两端同时优化纯 raw-input 链路：一方面升级 size-only router，使其在 Stage3 正窗上的 size top-1 提升至 0.6894；另一方面引入 route-aware 深度训练，让 depth expert 不只看到 ground-truth route，还在训练中显式适应 hard predicted route、soft route 与 top2 route。经过这两步后，纯 raw-input 路线的 predicted-route 平衡准确率提升至 0.5240，首次稳定超过 XGBoost 的 0.5138。

### 3.9 面向部署的统一层级反演器进一步提升了真实 predicted-route 条件下的系统级性能
针对上述瓶颈，本文进一步采用统一层级反演器，将大小判别、路由策略和深度专家放入同一训练图中协同优化。该模型不再只依赖 raw-input 学习，而是在部署模型中引入结构化物理特征分支，以提升 route-aware 鲁棒性。结果显示，该模型在测试集上的 ground-truth route 深度平衡准确率达到 0.5407，而在更关键的 predicted-route hard balanced accuracy 上达到 0.5337，不仅高于早期 raw predicted-route 链路的 0.4822，也高于优化后的纯 raw route-aware 链路的 0.5240，并继续稳定超过 XGBoost 的 0.5138。这里需要强调的是，这个结果主要服务于“最终系统是否更稳”这一工程问题，而不是再去证明“纯 raw-input 是否足够可学”。

### 3.10 实时界面验证了该体系具备术中系统化使用潜力
在工程层面，本文已将阶段一检测器与统一层级反演器集成到实时可视化界面中。界面可以同步显示原始触觉图、检测概率、大小分布和深度分布，并遵循“检测始终显示、大小与深度仅在高置信条件下输出”的逻辑。这一点说明本文并非只停留在离线模型比较，而是已经将“结构化机制发现 -> raw-input 学习 -> 可解释性验证 -> 部署增强 -> 术中显示”串接为一个可部署的系统原型。

### 3.11 延迟对比进一步说明，结构化特征提取限制了 XGBoost 作为实时主线的适用性
为了更客观地比较两条技术路线在实时场景中的适用性，本文进一步区分了两类延迟口径：一类是输入已经整理完成后的 `model-only` 推理时间，另一类是从原始 10 帧触觉窗口开始、包含归一化或特征整理的 `end-to-end` 延迟。结果显示，在纯 CPU 表格模型本体推理上，XGBoost 仍具有较低的单次预测开销；然而，一旦将窗口级物理特征提取纳入同一条推理链，其结构化路线的预处理负担便明显上升。以检测任务为例，XGBoost 路线的端到端中位延迟约为 27.34 ms，而 raw-input 检测神经网络在 CPU 上约为 7.11 ms，在 GPU 上约为 2.81 ms。该结果说明，尽管 XGBoost 作为离线结构化基线非常有价值，但“先提特征、再分类”的组织方式会增加实时系统中的工程开销和接口复杂度；相比之下，直接从原始时空窗口完成推理的神经网络更符合 detection-first 在线部署需求。需要同时指出的是，在当前 size/depth 的 CPU 结构化链路上，XGBoost 仍保持较快的端到端预测速度，因此本文不将“神经网络绝对更快”作为主结论，而将其更稳妥地概括为：神经网络在 detection 的 raw-input 端到端口径、系统级任务组织以及 predicted-route depth 性能上更具优势，而 XGBoost 更适合作为结构化机制参照。

## 4 讨论
本文最重要的结论不是“神经网络在所有指标上都压过了 XGBoost”，而是更具体也更符合临床需求的三点。第一，柔性触觉时空数据首先能够支持**稳定的结节探测**，这是最核心、也最具直接临床意义的结果。第二，在探测基础上，数据中确实还包含较强的大小信息以及较弱但可学习的深度信息。第三，深度信息的利用并不能通过一个平级多头网络自然获得，而必须通过显式的层级任务组织、大小感知路由以及对真实误差传播的直接优化来实现。

如果从方案价值来总结，本文真正扎实的地方主要体现在三个方面。其一，是问题拆分扎实。我们没有把所有任务硬塞进一个统一黑箱，而是根据数据中的信号强弱和物理耦合关系，先后定义 detection、size 和 coarse depth 三个层次，使模型结构与实验事实一致。其二，是证据组织扎实。我们没有直接以深度学习分数替代全部论证，而是先用机制分析和 XGBoost 证明信息存在，再用 raw-input 神经网络证明原始张量可学，最后再通过解释性分析去确认网络内部编码的结构。其三，是工程落地扎实。我们不仅比较离线指标，还把路由误差、端到端延迟和界面集成纳入分析，使“能不能上线到术中系统”不再只是一个口头承诺。

从这个角度看，XGBoost 与神经网络并不是竞争关系，而是递进关系。XGBoost 首先回答“数据里有没有可学习的信息、哪些物理特征在起作用”；raw-input 神经网络随后回答“在不直接输入这些手工特征的情况下，原始时空触觉窗口能否自动恢复这些信息”；而部署导向的统一层级反演器最后回答“在真实 route 误差存在时，如何把这些信息组织成更稳的系统性能”。因此，本文并不是“先试一个传统模型，再换一个更复杂模型”，而是依次建立机制认识、结构化基线、raw-input 学习、模型解释和部署增强五个层次的证据链。

需要特别强调的是，本文中的两类神经网络模型承担着不同职责。raw-input 层级神经网络是**科学主模型**，它的任务是证明原始时空张量本身可学，并通过解释性分析回答模型是否自动编码了与大小和深度相关的物理特征；统一层级反演器则是**部署主模型**，它的任务是利用结构化先验和 route-aware 优化，把系统在真实 predicted-route 条件下做得更稳。因此，如果不区分这两个层次，就会把“机制证明”和“系统增强”混为一谈，从而削弱文章主线的说服力。

本文还揭示了一个对术中系统设计非常重要的事实：单点的最佳中间任务指标，并不等价于系统级最优。以大小任务为例，raw-input size-only v2 已经在 standalone 指标上超过 XGBoost，而统一层级反演器在真实 predicted-route 深度上获得了更优结果。这表明，对于最终系统而言，更值得优化的不是某一个孤立任务的数值上限，而是整条推理链在真实误差传播条件下的鲁棒性。换言之，本文真正优化的对象不是“最好的 size 模型”或“最好的 depth 头”，而是“最可靠的 detection-first 术中决策链路”。

关于深度问题，本文当前的证据支持将其定位为**coarse depth discrimination**，而不是连续深度反演。一方面，XGBoost 和 raw-input size-routed 神经网络都已证明深度信息客观存在且可以被粗粒度辨别；另一方面，phase occlusion 和 hard-pair 分析也提示，当前 raw-input 模型虽然不再只是依赖峰值幅值，但在时间维上仍偏向 `peak neighborhood`，对 `loading/release` 阶段的利用仍不足。因此，深度相关结果在主文中应主要承担“给出证据”的职责，而它为何仍然困难、为何尚未完全恢复全部物理机制，则更适合在 Discussion 中解释和界定边界。

本研究仍存在若干局限。第一，当前评估协议本质上属于 repeat-held-out 验证，而非更严格的 leave-condition-out 泛化评估；因此，本文更适合将当前结果解释为在固定物理条件设计矩阵下的稳健验证，而非对更广泛临床域泛化能力的最终证明。第二，本文的深度任务目前仍是三类 coarse depth classification，而非更精细的连续深度定量。第三，尽管解释性分析已支持 raw-input 神经网络确实自动编码了部分与大小和深度相关的结构，但这些结果仍应被理解为“部分机制一致性证据”，而非对完整生物力学规律的最终恢复。第四，统一层级反演器虽然在 predicted-route 深度上取得了最佳部署结果，但由于其引入了结构化物理特征分支，因此它更适合被解释为系统增强模型，而不应被单独用作“raw-input 自动学习机制”的唯一证据。未来工作需要继续强化 phase-aware 建模、设计更针对 routing noise 的训练策略，并在更接近真实临床场景的数据上进一步验证系统的泛化能力。

与既有微创触觉定位文献相比 [R4-R8]，本文更希望强调的是问题组织方式上的推进，而不是简单宣称性能超越所有已有工作。已有研究已经证明触觉感知可用于包块定位、机器人辅助手术力反馈和新型传感器设计；而本文进一步补上的，是把这一方向放回术中肺结节这一明确场景下，系统回答“先探测、再大小、后粗深度”的任务层级，并用结构化基线、raw-input 学习和可解释性分析共同支撑这一组织方式的合理性。因此，本文更适合作为一项把触觉传感、时空建模与系统验证重新组织起来的 biomedical engineering 探索，而不是把自己写成对全部已有触觉研究的终局性替代。

## 5 结论
本文围绕术中肺结节触觉缺失问题，提出并初步验证了一条以**可靠探测为核心**、逐步扩展到大小反演和粗深度辨别的研究路径。结果表明：柔性触觉时空数据首先能够支持稳定的结节探测；在此基础上，结节大小可被较可靠地反演，而深度信息则作为更弱、更依赖层级组织的条件信号被粗粒度辨别。通过机制分析和 XGBoost 基线，本文先证明了这些信息的存在与可学习性；通过 raw-input 层级神经网络，进一步证明原始时空输入本身可以在不直接依赖结构化物理特征的情况下学习到这些信息；通过可解释性分析，则验证神经网络内部确实自动编码并利用了部分与大小和深度相关的物理结构；最后，通过统一层级反演器，本文又将这些认识进一步转化为真实 predicted-route 条件下更稳的部署性能。整体而言，本文更接近于提出一个“检测优先、反演增强、可解释验证、部署增强分层组织”的术中触觉定位原型框架，而不是给出已经完全定型的临床终态系统。这一探索为柔性触觉感知在术中肺结节定位中的进一步工程化和临床验证提供了较为扎实的技术起点。
