# 论文图表重置方案（TBME 版）

## 为什么要重置
前一轮图表探索暴露了一个问题：
- 有些图在技术上可以生成，但证据力不足；
- 有些图视觉上不够稳，反而削弱主文说服力；
- 有些图虽然“有意思”，但并不能直接回答 TBME 审稿人最关心的问题。

因此，当前图表策略重置为：
**只保留能直接支撑主文结论的图。**

## 当前结论
### 明确删除
- 各类弱证据表征空间图
- 各类已淘汰的对照拼图
- 任何不能一眼说明问题的探索性图

### 明确保留
- 引言总图
- 方法总图
- 机制分析 + XGBoost 解释
- detection 结果
- size 结果
- raw-input depth 主结果
- raw-input explainability 主图
- deployment enhancement
- GUI 原型图
- 延迟 benchmark

## 当前图表主线
1. 为什么术中肺结节定位难
2. 系统如何从触觉实验走到层级模型
3. 数据里到底有哪些可学信息
4. raw-input 神经网络是否在主任务上成立
5. raw-input 神经网络是否自动编码了部分物理结构
6. 为什么最终系统仍需要部署增强
7. 系统是否具备实时原型形态

## 当前图表禁区
1. 不能让图承担超出其证据能力的论证。
2. 不能用“好看但说不清”的图占据主文位置。
3. 不能用弱证据表征图替代统计结果图。
4. 不能为了可视化多样性而牺牲论文主线清晰度。
