# 基于柔性触觉感知与深度学习推理的术中肺结节定位探索研究

## 摘要
**目的**：胸腔镜和机器人辅助手术减少了创伤，却同时削弱了术者对肺组织的直接触觉判断，使小结节、深部结节和表面不显著结节的术中定位更加困难。乳腺等浅表组织触诊研究已经证明，内部病灶能够通过表面机械响应被感知；但肺又是含气、易塌陷且术中持续变形的器官，术前 CT 与术中真实组织状态之间存在明显失配。本文旨在围绕这一临床痛点，探索柔性触觉感知能否从浅表触诊场景迁移到术中肺结节定位，并进一步回答原始触觉时空信号是否支持结节探测、大小反演和粗深度辨别。

**方法**：本文构建了一套由柔性阵列触觉传感器、前端采集单元和计算工作站组成的术中触觉原型系统，并设计围绕真实猪肺组织与仿体结节的重复实验，形成了覆盖 7 个结节大小、6 个埋藏深度和 3 次重复的离体触觉实验数据集。研究首先结合相关文献、触觉机制分析与结构化特征提取，采用 XGBoost 逐层验证 detection、size 和 coarse depth 的可学性，并利用 SHAP 识别各任务依赖的关键物理特征；在此基础上，再构建仅输入原始时空张量的层级神经网络，按 `Detection -> Size -> Depth` 的顺序开展学习，并通过 latent probe、Integrated Gradients、hard-pair analysis 和 phase occlusion 对深度模型进行解释性分析。最终，部署增强模型被整合进实时 GUI。

**结果**：结构化基线首先表明，柔性触觉数据中存在稳定的结节存在信息、较强的大小信息以及更弱但可学习的粗深度信息。raw-input 检测模型在独立测试集上取得 `AUC = 0.8383`，高于 XGBoost 的 `0.8199`；raw-input size-only 模型取得 `Top-1 = 0.7177`、`Top-2 = 0.8195` 和 `MAE = 0.1242 cm`，全面优于 XGBoost 的 `0.6701`、`0.8023` 和 `0.1472 cm`。在深度任务上，普通共享深度头会退化至多数类附近，而在 `size-aware + route-aware` 组织下，纯 raw-input 路线的 predicted-route 平衡准确率达到 `0.5240`，超过 XGBoost 的 `0.5138`；部署增强模型进一步提升至 `0.5337`。解释性分析显示，当前 route-aware 深度模型的 latent probe 平均测试 `R^2 = 0.2712`，高于 size-only probe 的 `0.2356`；Integrated Gradients、hard-pair 和 phase occlusion 共同表明，模型已利用部分与深度相关的空间结构，但时间策略仍偏向 `peak neighborhood`。

**结论**：柔性触觉时空信号能够首先支持可靠的术中结节探测，并在此基础上支持较稳定的大小反演；对更弱且更耦合的深度变量，则需要在大小条件下进行更保守的粗粒度辨别。本文的意义不在于宣称已经完成一个定型临床系统，而在于建立了一条从场景迁移、重复实验和数据构建，到结构化验证、raw-input 学习、解释性确认与系统部署的完整技术路径。

**关键词**：肺结节探测；柔性触觉阵列；术中导航；大小反演；粗深度分类；可解释人工智能

## 1 引言
低剂量 CT 筛查的普及使临床上对微小肺结节和磨玻璃结节的检出率持续上升。对于这类疑似早期病灶，胸腔镜和机器人辅助手术因创伤小、恢复快而被广泛采用。然而，微创获益伴随着一个长期悬而未决的问题：术者在细长器械辅助下操作时，无法像开放手术那样直接触摸肺组织，从而失去了对结节位置、边界和埋藏情况的重要触觉判断。对于深部、小尺寸或表面不明显的结节，这种触觉缺失会显著增加术中定位难度。

目前常用的辅助方案包括术前 CT 引导钩线或染料定位、术中超声以及有限度的数字触诊等 [R1-R3]。这些方法各自具有临床价值，但也分别面临侵入性风险、空气干扰、经验依赖强或定量性不足等局限。换言之，术中定位真正缺失的并不只是更多视觉信息，而是一条能够在微创条件下恢复机械触感、并将其转化为客观判断的技术路线。

本文并不是凭空提出用触觉做肺结节定位。前人在乳腺等浅表组织触诊、柔性电子皮肤成像以及微创肿瘤触觉定位方向的研究已经证明，只要组织内部存在局部力学异质性，有限按压就会在表面留下可重复的机械响应变化；这些变化不仅表现为单点峰值差异，还会体现在扩散范围、形态对比和时间演化上 [R4-R8]。这一点为本文提供了最核心的可迁移依据：**无论是乳腺中的浅表肿块，还是肺组织中的埋藏结节，本质上都属于“内部高刚度包块改变表面触觉响应”的问题**。换句话说，能够从乳腺等浅表组织迁移到肺，并不是因为器官相似，而是因为它们共享“内部机械异质性 -> 表面时空响应变化”这一更底层的物理机制。

但这种迁移又绝不是直接照搬。乳腺等浅表触诊通常面对的是相对静止、表面可达、组织参照更稳定的场景，而肺结节定位面对的是含气、柔软、易塌陷、曲率变化明显且术中持续变形的器官。两者的相似性在于：都需要通过局部按压把深部病灶转换成表面可观测响应；两者的差异性在于：肺部场景中，患者体位变化、麻醉、单肺通气和术中肺萎陷都会使术前静态 CT 与术中真实组织形态之间出现明显失配，结节深度效应还会与大小、扩散、形态和按压阶段发生更强耦合。也就是说，肺结节定位不仅是“能不能感知包块”的问题，更是一个**如何在 CT 参照不再稳定可靠时，用术中实时触觉补足定位判断**的问题。本文的第一项核心贡献，正是在承认这种差异性的前提下，把原本主要服务于浅表肿块触诊的柔性触觉感知范式迁移到术中肺结节定位，并围绕这一问题搭建完整实验系统。

前人确实已经做过和本文相关的重要工作，但它们更多解决的是“触觉能否帮助发现病灶”或“设备能否在微创条件下恢复部分触感”这类问题，而不是本文要回答的整套递进问题。已有微创触觉定位研究已经证明，深部包块的存在能够在动态接触中留下可学习痕迹，也因此为 `detection-first` 提供了直接先验 [R4-R8]；但这些研究大多停留在 detection 或 localization，没有在同一套触觉时空信号中继续系统追问 **size** 和 **depth**。此外，不少工作建立在更理想化的组织模型、单一病灶任务或设备原理验证之上，较少处理肺组织这一更柔软、更易变形器官中的 `size-depth` 耦合问题，也较少提供“结构化特征验证 -> raw-input 学习 -> 可解释性验证 -> 部署增强”的完整证据链。因此，对本文而言，可直接迁移过来的主要是 detection-first 的研究起点、动态按压优于静态单帧的经验，以及“内部病灶会改变表面时空响应”的物理直觉；而本文真正补上的，不只是 detection 之后的大小与粗深度问题，更是**把触觉从经验性触诊推进到定量化推理**。

这也是本文与常见“多头并列输出”写法最不同的地方。我们的实验设计天然包含两个真实物理变量：结节大小和结节深度。但从研究问题的形成过程来看，首先浮现出来的并不是这两个变量本身，而是更直接、也最符合前人工作基础的临床问题：**当前位置下方是否存在结节**。换句话说，本文之所以先做 detection，并不是因为它“简单”，而是因为：
1. 这是已有微创触觉定位研究最成熟、最能直接迁移过来的起点；
2. 这是临床使用中最直接的第一需求；
3. 只有当 detection 成立以后，继续追问大小和深度才有意义。

只有在这个问题成立以后，才值得继续去问：如果不同大小的结节在动态触觉分布上存在稳定差异，那么这些差异是否足以支持**大小反演**。而深度问题则更晚出现，因为从机制上看，深度效应通常弱于大小效应，而且与大小、扩散、形态和时序阶段等多种因素耦合。也正因为如此，本文最终没有把深度定义为连续回归，而是更谨慎地把它界定为 `coarse depth discrimination`。

因此，本文不是从“检测、大小、深度三层任务”出发去寻找一个统一黑箱模型，而是采用一条更符合问题自然展开过程的研究路径：
1. 先确认柔性触觉时空信号能否支持可靠结节探测；
2. 在探测成立之后，再判断信号中是否存在可稳定反演的大小信息；
3. 在大小和深度同时存在的实验矩阵下，最后才谨慎进入更弱的粗深度问题。

这条路径直接决定了本文的方法组织。我们首先根据文献、机制分析和原始数据观察提出候选结构化物理特征，并使用 XGBoost 逐层验证 detection、size 和 coarse depth 是否可学，同时通过 SHAP 识别不同任务真正依赖的关键特征家族。只有在这一结构化证据已经建立之后，我们才进一步构建仅输入原始时空张量的层级神经网络，去回答“原始输入本身能否自动学习这些逐步发现的结构”。这一点对应本文的第二项核心贡献：**将深度学习正式引入触觉感知推理，使系统能够在结节是否存在之外，进一步对大小和粗深度进行量化判断**。最后，再通过解释性分析检验网络是否确实利用了与结构化基线一致的物理线索，并通过部署增强模型与 GUI 验证其工程意义。

因此，本文真正想强调的不是“我们也做了 detection”，而是“为什么 detection 必须是第一步，以及 detection 之后哪些问题是前人没有继续做下去、而本文继续往前推进的”。可以直接从前人工作中迁移过来的，是 detection-first 的问题定义、病灶机械对比会留下表面响应痕迹这一基本物理直觉，以及动态按压优于静态单帧的经验；而本文补上的，是在肺结节场景下用统一实验矩阵和统一证据链把 detection、size 和 coarse depth 串联起来，并进一步用 raw-input 学习和解释性分析把这条路线真正做完整。

在这个框架下，XGBoost 并不是一个等待被替换的旧方法，而是本文证据链中的第一个关键环节。它首先回答“任务是否可能”；随后，raw-input 神经网络才去回答“这些信息能否直接从原始时空张量中被学到，而且能否更好地被学到”。这种安排使本文的逻辑不再是“先有三个任务，再上两个模型去比较”，而是“先有一个逐步展开的问题，再用不同层级的方法去验证这个问题”。

基于上述认识，本文希望回答以下四个问题：
1. 柔性触觉时空数据能否首先支持稳定可靠的术中结节探测？
2. 如果探测成立，这些信号是否还包含较强且可反演的大小信息？
3. 对于更弱且更耦合的深度变量，是否能够在更保守的 coarse depth 层面获得可辨识性？
4. 当 raw-input 神经网络达到甚至超过结构化基线之后，它内部是否确实编码了与这些物理特征一致的关键结构？

围绕这些问题，本文的主要贡献可以概括为：
1. **场景迁移创新**：将原本更多用于乳腺等浅表肿块触诊与成像的柔性触觉感知范式迁移到术中肺结节定位场景，明确阐明两者共享的底层物理基础与器官层面的关键差异，并针对术前 CT 与术中肺组织状态失配这一关键难题，构建了一套由柔性阵列传感器、采集单元和计算工作站组成的术中触觉原型系统。
2. **数据集与实验创新**：设计了围绕真实猪肺组织与仿体结节的重复实验流程，建立了一个覆盖 7 个大小、6 个深度和 3 次重复的面向临床场景触觉实验数据集，并形成统一的窗口级标注与评估协议。
3. **定量推理创新**：在结构化物理特征和 XGBoost 基线验证可学性的基础上，将深度学习引入触觉感知推理，提出沿 `Detection -> Size -> Depth` 顺序展开的 raw-input 层级神经网络，使触觉信息从“能否探测到结节”的定性判断，扩展到对结节大小和粗深度的量化推理，并在关键任务上达到甚至超过 XGBoost。
4. **解释性闭环与系统落地**：通过 SHAP、latent probes、Integrated Gradients、hard-pair analysis 与 phase occlusion，说明网络确实自动编码了与关键物理因素一致的部分结构，而不是仅依赖单点峰值捷径；同时构建统一层级反演器与实时界面原型，形成面向实验人员和潜在临床使用者的完整触觉定位与推理工作流。

## 2 材料与方法

### 2.1 问题是如何逐层形成的
本文首先面对的并不是一个“三层反演”定义好的学习任务，而是一个临床与实验共同驱动的问题形成过程。

第一层问题是**结节是否存在**。这是微创术中最直接、最基础的定位需求，也是最先值得被验证的问题。

第二层问题是**大小能否被反演**。当实验矩阵显示不同大小条件下的触觉分布存在系统性差异时，大小问题才自然出现；它不是一个独立追加的任务，而是从 detection 成立后进一步延伸出来的问题。

第三层问题是**深度是否具备可辨识性**。深度变量虽然在实验设计中客观存在，但它的效应并不如大小直接和稳定。因此，本文并不把深度作为一开始就与前两者并列的目标，而是在前两层证据建立之后，才将其谨慎定义为 `shallow / middle / deep` 的粗分类探索问题。

### 2.2 硬件系统、实验平台与数据协议
本文的实验系统由三部分组成：`(1)` 与肺表面接触的柔性阵列式触觉传感器，`(2)` 负责将阵列信号连续读出的前端采集单元，以及 `(3)` 用于数据记录、可视化和后续分析的计算工作站。该系统的硬件角色不是在术中直接给出最终判断，而是尽可能稳定地把肺表面的局部力学响应转化为可分析的时空触觉序列。

柔性阵列贴附或轻压于肺表面时，埋藏结节与周围肺实质之间的力学不匹配会改变表面应力分布。系统每一帧记录 96 个传感通道，随后重排为 `12 × 8` 的二维应力矩阵。与只读取单点峰值的方案不同，这里保留的是一个随动态按压连续变化的二维表面响应场，因此既能描述局部峰值，也能保留扩散、形态和阶段变化信息。

实验平台采用离体猪肺模型，并在其中埋藏预设尺寸和深度的仿体结节。整个实验流程与前期技术交底书中的动物实验流程保持一致，即：完成结节造模与埋藏后，将柔性阵列置于目标区域表面，执行连续动态按压，实时记录完整触觉序列，并在工作站侧完成可视化与数据保存。围绕这一流程，本文进一步设计了标准化重复实验方案，以保证不同大小、不同深度和不同重复之间具备可比性。本文沿用这一实验流程，但算法分析部分已完全切换为当前论文中的 `XGBoost baseline -> raw-input hierarchical neural network -> explainability -> deployment enhancement` 主线，而不再使用交底书中的旧检测算法。

最终形成的数据集覆盖 7 个结节大小和 6 个埋藏深度，共 42 个物理条件；每个条件进行 3 次独立重复，共获得 126 组带完整时间演化信息的触觉记录。严格来说，这是一套**面向临床定位场景的实验数据集**，而不是患者临床数据集；其价值在于用真实猪肺组织与仿体结节组合，尽可能逼近术中触诊的关键力学环境。模型采用长度为 10 帧、步长为 2 的滑动窗口作为基本样本单位。检测任务的标签由人工标注阳性片段决定，窗口中心帧位于阳性片段时记为正窗；大小和深度标签则直接继承自实验条件。检测任务采用 `1.CSV + 2.CSV` 开发、`3.CSV` 测试的协议；大小与深度任务采用更严格的 `1.CSV` 训练、`2.CSV` 验证、`3.CSV` 测试协议，并仅在真实阳性窗口上训练。

### 2.3 第一层：先用结构化特征回答“能不能探测到结节”
在深度学习之前，本文首先从原始触觉序列中提取具有物理意义的结构化特征。其来源由三部分共同决定：
1. 现有微创触觉与病灶定位文献对机械响应的启发；
2. 本实验中大小和深度变量对表面应力场影响的机制判断；
3. 原始数据中可稳定观察到的峰值、扩散、形态和阶段响应差异。

最终使用的特征大致可归纳为五类：`amplitude response`、`spread extent`、`shape contrast`、`deformation position` 和 `temporal phase`。在第一层问题上，XGBoost detection baseline 主要用于验证：这些结构化特征是否足以支持稳定的结节存在判断。

### 2.4 第二层：在探测成立后，再回答“能不能反演大小”
当前述结构化分析表明不同大小条件之间存在稳定差异后，本文才正式将大小定义为第二层问题。为此，XGBoost 被进一步用于 size classification 和 regression，以验证大小信息是否能够从同一套触觉时空信号中被反演。与此同时，SHAP 被用来识别大小任务最依赖的特征家族，从而判断大小是一个强主效应还是偶然相关性。

### 2.5 第三层：最后才谨慎进入粗深度问题
只有在 detection 和 size 两层证据都已建立之后，本文才将深度作为一个后续问题引入。考虑到深度效应更弱且与大小强烈耦合，本文不将其建模为连续回归，而是采用 `shallow / middle / deep` 三类粗分类。此时，XGBoost 的作用不是追求最终最优，而是先回答：在结构化特征空间中，深度是否至少在粗粒度层面可学。

### 2.6 raw-input 层级神经网络：沿着同样的顺序重新学习问题
在结构化基线逐层证明任务可行后，本文进一步构建仅输入原始时空张量的层级神经网络，并沿着完全相同的顺序展开：

#### 2.6.1 阶段一：结节探测
阶段一检测器直接从原始时空触觉窗口判断结节是否存在。输入包括原始幅值窗口和相邻帧差分窗口，空间特征由轻量卷积编码，时间特征由 MS-TCN 聚合，最后输出结节存在概率。其目标是先建立稳定可靠的 detection mainline。

#### 2.6.2 阶段二：大小反演
在真实阳性窗口上，本文进一步训练 raw-input size-only router v2，同时输出 7 类大小分类和连续大小估计。该模型采用 `size classification + ordinal size prediction + expectation-based continuous estimation + residual correction` 的组合结构，以同时兼顾档位判断和档内连续修正。由于大小是强主效应，同时又是深度建模的路由条件，因此它被单独建模，而不是简单附属于共享多任务头。

#### 2.6.3 阶段三：大小条件下的粗深度分类
深度任务并未被建模为一个普通共享头，而是被组织为 `size-aware` 的 routed experts。其核心原因是：深度效应本身较弱，而且会被大小这一更强因素掩盖。如果仍要求网络在全局共享特征上直接分出深度类别，模型往往会优先吸收大小和强度差异，导致深度退化。为此，本文将深度建模为在大小条件下进行的 `shallow / middle / deep` 三分类，并进一步通过 `GT route / hard predicted route / soft route / top2 route` 的联合训练实现 `route-aware` 优化。

### 2.7 可解释性分析
为了验证 raw-input 神经网络是否真正学到了与大小和深度相关的结构，本文对当前 route-aware depth v2 模型执行四类解释性分析：
1. **latent linear probes**：在冻结模型后，用潜在表示恢复前述结构化特征，检验网络是否自动编码了这些概念；
2. **Integrated Gradients**：在正确预测测试窗口上计算输入归因，并按深度类别聚合为平均空间归因图；
3. **hard-pair analysis**：构造“大而深 / 小而浅”等容易混淆的样本对，观察模型是否仍能在减弱峰值差异后给出更合理判断；
4. **phase occlusion**：按 `loading early / loading late / peak neighborhood / release` 分阶段遮挡输入，检验模型在时间维上真正依赖什么。

### 2.8 部署增强与 GUI
在 scientific main line 之外，本文还构建了部署导向的统一层级反演器。该模型联合输入原始幅值、归一化形态、差分序列和结构化物理特征，并同时优化 size 与 routed depth，以增强真实 predicted-route 条件下的系统鲁棒性。最终，阶段一检测器与统一层级反演器被整合到实时 GUI 中，界面持续显示 detection 结果，而 size 与 depth 仅在高置信条件下输出。

## 3 结果

### 3.1 我们首先问的是：结节能否被可靠探测到
结构化特征分析首先表明，在结节存在与否这个问题上，触觉信号中的差异最为显著。XGBoost detection baseline 在测试集上取得 `AUC = 0.8199`，说明原始触觉时空信号中首先存在稳定的结节存在信息。这个结果的重要性在于，它让后续所有问题都有了前提：如果连结节是否存在都无法被稳定判断，那么大小和深度问题就没有继续展开的基础。

进一步地，阶段一 raw-input 检测器在独立测试集 `3.CSV` 上取得 `AUC = 0.8383`、`AP = 0.5357` 和 `F1 = 0.6216`，整体优于 XGBoost 的 `0.8199`、`0.5063` 和 `0.6185`。也就是说，沿着这一第一层问题继续深入，原始时空张量本身已经能够支撑比结构化 baseline 更强的 detection。

### 3.2 在探测成立之后，我们才进一步问：大小信息是否存在
当 detection 的可学性被确认后，结构化特征分析进一步显示，不同大小条件下的触觉分布存在稳定差异。XGBoost 在大小任务上取得 `Top-1 = 0.6701`、`Top-2 = 0.8023` 和 `MAE = 0.1472 cm`，说明大小信息不是偶然噪声，而是能够被稳定反演的强主效应。

沿着同一问题继续推进，优化后的 raw-input size-only router v2 在真实阳性窗口上达到 `Top-1 = 0.7177`、`Top-2 = 0.8195` 和 `MAE = 0.1242 cm`，全面优于 XGBoost。这一步说明，当问题被自然地从 detection 延伸到 size 时，原始时空张量本身已经足以支撑比结构化 baseline 更好的大小学习。

### 3.3 只有在前两层问题成立之后，深度才成为一个值得提出的问题
与 detection 和 size 不同，结构化特征分析并没有表明深度是一个同样强的独立主效应。相反，深度表现为更弱且容易被大小掩盖的条件依赖信号。即便如此，XGBoost 在粗深度任务上仍取得 `balanced accuracy = 0.5138`，显著高于多数类基线 `0.3333`。这个结果的意义不在于“深度已经被很好解决”，而在于它首次说明：在更保守的 coarse depth 层面，深度至少是有可能被学习到的。

### 3.4 深度之所以难，不是因为信息不存在，而是因为不能把它当成并列共享任务
最初的普通 raw-input 深度头在严格重复划分协议下退化到多数类附近，说明深度任务不能被当作一个普通共享头直接处理。当深度改写为 size-routed experts 后，ground-truth route 的平衡准确率首先提高到 `0.5238`，略高于 XGBoost 的 `0.5138`。在进一步引入 route-aware 训练后，当前 route-aware depth v2 的 `GT-route balanced accuracy = 0.6066`，而更关键的 `predicted-route balanced accuracy = 0.5240`，也首次稳定超过 XGBoost 的 `0.5138`。这说明：深度并不是不可学，而是必须在“大小已知或大小被先估计”的语境下被组织起来。

### 3.5 可解释性结果进一步说明：网络确实学到了我们前面逐层发现的那些结构
在当前 route-aware depth v2 模型上，latent probe 的平均测试 `R^2 = 0.2712`，高于 size-only probe 的 `0.2356`。这说明模型内部并不只是保留了大小标签，而是编码了部分与深度相关的更细结构。进一步地，可恢复性较高的概念包括 distribution complexity、shape contrast、amplitude response 和 rise-time-to-peak，表明网络学到的并不是单一捷径。

Integrated Gradients 进一步显示，在每个深度类别各选取 40 个正确预测测试窗口后，`shallow / middle / deep` 三类对应的平均空间归因区域并不完全相同，模型在空间上已经形成了与类别相关的关注结构。与此同时，hard-pair analysis 表明，在构造的困难样本对中，更深样本在约 `75%` 的配对中获得更高的 `p_deep`，说明模型并非单纯依赖峰值幅值完成判断。

不过，phase occlusion 也给出了一个同样重要的反向信息：遮挡 `peak neighborhood` 时，真实类别概率平均下降达到 `0.2303`，明显高于 `loading early (0.0662)` 与 `release (0.0446)`。这意味着当前模型虽然已经自动编码了部分与深度相关的物理结构，但其时间策略仍偏向峰值邻域，对完整时序机制的利用仍不充分。

### 3.6 因而，本文真正建立的是一条渐进式证据链
把上述结果连起来看，本文建立的不是一个“三层任务同时成立”的静态结论，而是一条更符合问题自然展开方式的证据链：
1. 首先证明，原始触觉信号可以支持可靠结节探测；
2. 然后证明，在探测成立后，同一信号中还存在可稳定反演的大小信息；
3. 最后才谨慎表明，在大小条件下，更弱的粗深度信息也能够被学习；
4. 再通过解释性分析确认，神经网络内部确实利用了与这些逐层发现的物理结构一致的线索。

### 3.7 部署增强模型与实时系统验证了工程可行性
尽管 pure raw route-aware 模型已经在 predicted-route 深度任务上超过 XGBoost，但真实部署仍然受 `size -> depth` 路由误差影响。为此，本文进一步构建统一层级反演器，在联合原始时空输入与结构化特征后，将 predicted-route 深度平衡准确率进一步提升至 `0.5337`。最终，该体系被部署到实时 GUI 中，形成 detection-first 的术中触觉导航原型。系统持续显示 detection 结果，并仅在高置信条件下释放 size 和 depth 输出，从而更符合实际使用场景。

## 4 讨论
本文最核心的改进，并不是简单提出了一个新的神经网络，而是完成了三件更本质的事情。第一，是把柔性触觉感知从乳腺等浅表肿块触诊场景迁移到术中肺结节定位场景，回应了术前 CT 与术中真实肺组织状态失配这一长期存在但难以通过传统导航彻底解决的问题。第二，是围绕真实猪肺组织与仿体结节设计重复实验，构建出一套能够支撑算法验证和系统比较的统一实验数据集。第三，是把深度学习真正引入触觉感知推理，使系统不再停留在“有没有结节”的经验判断，而能够进一步对大小和粗深度给出量化估计。围绕这三件事，本文才进一步把术中肺结节触觉定位问题组织成了一条更自然的研究路径：先探测，再大小，最后才谨慎进入深度。

从方法论上看，本文采用的“机制分析与结构化 baseline 先行，再进入 raw-input 深度学习，最后用可解释性闭环验证”的范式，并非多余步骤，而是当前这类 biomedical engineering 问题中更稳健的做法。一方面，它避免了把全部结论都压在单一黑箱模型上；另一方面，它允许我们先回答“信息是否存在”，再回答“原始输入是否足以自动学习这些信息”。这对样本规模有限、问题带有明显物理耦合和解释性要求的触觉场景尤其重要。

关于深度问题，本文的结果支持一个相对克制但清晰的结论：深度不是不可学，但也远未达到“连续深度反演”这一更强主张。XGBoost 与 raw-input 路线都表明深度信息客观存在，而 route-aware raw model 在 predicted-route 条件下也已超过 XGBoost；然而，phase occlusion 同时说明模型仍明显依赖 peak neighborhood，这意味着当前网络尚未充分利用 loading / release 等更完整的时序信息。因此，本文关于深度的定位应当是**探索性的粗粒度辨别**，而不是已经解决的精确反演。

本文也存在几个需要明确界定的边界。第一，当前评估协议本质上是基于重复实验的 `repeat-held-out` 验证，而不是更严格的 `leave-condition-out` 泛化评估，因此结果应主要解释为在固定物理设计矩阵下的稳定验证。第二，当前 explainability 结果已经能支持“网络确实学到了部分物理结构”这一结论，但并不能说明网络已经完整复现了前述机制分析中涉及的全部概念。第三，当前系统原型已具备工程可用性，但距离真实临床环境下的广泛验证仍有明显距离。

尽管如此，本文仍然给出了一个相对扎实的起点：柔性触觉时空数据不仅能够支持结节探测，还可以进一步支持大小反演和粗深度辨别；而且这种能力既可以通过结构化特征被验证，也可以由 raw-input 神经网络直接学到，并通过解释性分析得到反向支撑。更重要的是，本文已经把“触觉感知 -> 量化推理 -> 实时显示”串成了一个完整原型系统，为后续面向更复杂组织条件、更严格泛化协议和临床环境的持续研究提供了清晰方向。

## 5 结论
本文围绕术中肺结节触觉缺失问题，完成了三项相互支撑的核心创新。第一，本文将原本主要用于乳腺等浅表肿块触诊的柔性触觉感知范式迁移到术中肺结节定位，针对术前 CT 与术中肺组织状态失配这一难题，建立了实时触觉获取、分析和显示的原型系统。第二，本文设计了围绕真实猪肺组织与仿体结节的重复实验，形成了面向临床场景的统一触觉实验数据集，为后续模型训练、比较和解释提供了共同基础。第三，本文将深度学习正式引入触觉感知推理，在结构化基线验证可学性的基础上，实现了对结节存在、大小和粗深度的逐层量化推理。结果表明，柔性触觉时空数据能够首先支持稳定的结节探测，并进一步支持大小反演和粗深度辨别；同时，当前 raw-input 神经网络在 detection、size 和 predicted-route depth 等关键任务上已达到甚至超过结构化基线。整体上，本文更接近于提出一个“触觉迁移 + 数据集构建 + 定量推理 + 系统落地”的术中肺结节触诊框架，而不是宣称已经完成一个完全定型的临床终态系统。
